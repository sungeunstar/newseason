핵심 아이디어

"제안서를 쓰는 게 아니라,
이미 만들어서 링크로 보낸다"

콜드메일이 잘 안 읽히는 건, 받는 사람 입장에선 별로 볼 게 없기 때문이에요. "이렇게 개선하면 어떨까요"라는 말은 결국 상상에 기대는 설득이고, 바쁜 사람이 상상하며 답장을 보낼 이유는 없어요.

그래서 순서를 바꿨어요. 메일을 쓰기 전에 먼저 리디자인 페이지를 만들어요. GitHub Pages에 배포하고, 그 링크를 본문에 넣어서 보내요. 받는 사람은 자기 업체 이름이 들어간 완성된 페이지를 바로 보게 돼요.

Core Insight

Redesign

메일 보내기 전에
이미 그 업체 사이트를 만들어뒀어요

핵심은 받는 사람이 링크를 열었을 때 자기 업체 이름이 들어간 완성된 페이지가 보인다는 거예요. 임의로 만든 샘플이 아니에요. 업체 사이트를 크롤링해서 실제 텍스트(업체명, 소개, 서비스 항목)를 가져오고, Unsplash API로 업종에 맞는 이미지를 골라서 HTML 랜딩페이지를 만들어요.

만들어진 HTML은 GitHub Pages에 자동 배포돼서 공개 URL이 생겨요. 그 링크를 메일에 넣어서 보내요.

열람

How It Works

크롤링부터 배포까지,
4개 모듈이 자동으로 연결돼요

리디자인 파이프라인은 4개의 독립 모듈로 나뉘어요. crawler.py가 업체 사이트에서 텍스트를 추출하면, Unsplash API가 업종 키워드로 이미지를 선택해요. redesigner.py가 콘텐츠를 HTML 템플릿에 주입하고, deployer.py가 GitHub API로 Pages에 올려서 공개 URL을 만들어요.

모듈을 분리한 건 각 단계를 독립적으로 테스트하기 위해서였어요. 크롤링이 실패해도 이미 수집된 데이터로 리디자인을 돌릴 수 있고, 배포 단계에서 오류가 나도 생성된 HTML 파일은 로컬에 남아요.

Redesign 플로우

01

crawler.py — 업체 콘텐츠 추출

업체 사이트에서 업체명·소개·서비스·연락처를 파싱. 품질 점수 계산 후 낮은 곳만 선별.

02

Unsplash API — 업종 맞춤 이미지 선택

업종 키워드(clinic / interior / cafe 등)로 Unsplash에서 고품질 이미지를 자동 선택.

03

redesigner.py — HTML 랜딩페이지 생성

추출된 콘텐츠 + 이미지를 HTML 템플릿에 주입. 업체별 개인화된 랜딩페이지 생성.

04

deployer.py — GitHub Pages 자동 배포

생성된 HTML을 GitHub API로 push → 공개 URL 확보. 이 링크가 메일에 들어감.

Pipeline

자동으로 돌아간다

리디자인은 파이프라인의 가운데에 있어요. 키워드를 입력하면 수집, 크롤링, 리디자인, 배포, 발송이 순서대로 자동으로 실행돼요. 사람이 개입하는 구간은 없어요.

Journey — PEACH 2까지

LIME부터 v1, v2를 거쳐
지금의 구조가 됐어요

PEACH가 지금 형태가 된 건 처음부터 계획한 게 아니에요. 먼저 LIME이라는 실험 프로젝트가 있었고, 거기서 리디자인 자동화가 가능하다는 걸 확인한 뒤에야 본격적으로 시작할 수 있었어요.

LIME
선행 실험

자동 리디자인이 가능하다는 걸 처음 증명

Unsplash API와 `redesigner.py`를 처음 만든 프로젝트예요. 업종 키워드로 이미지를 자동 선택하고 HTML을 생성하는 방식을 여기서 검증했어요. output/ 폴더에 28개 이상의 실제 결과물이 쌓였어요. 이 코드베이스가 PEACH 리디자인 로직의 기반이 됐어요.

v1
첫 번째 버전

네이버 지역검색 기반, 파이프라인 첫 완성

LIME의 리디자인 로직을 가져와서 수집→크롤링→리디자인→발송 전체 파이프라인을 처음 완성했어요. 네이버 지역검색으로 병원·학원·인테리어 업체를 타겟으로 했어요. 작동은 됐지만 HTML 템플릿이 단조롭고 타겟이 너무 광범위했어요.

v2
타겟 전환

Cafe24 쇼핑몰로 전환, 퍼널 세분화

Cafe24 쇼핑몰이 개선 포인트가 더 명확했어요. 전환율 개선(퍼널A)과 해외 확장(퍼널B)으로 메일 내용을 나눴어요. v1과 v2가 별도 코드베이스로 분리되면서 관리가 어려워졌어요.

Unified ⭐
현재

LIME + v1 + v2 통합, 버그 12개 수정, PEACH.exe 완성

흩어진 코드베이스를 하나로 합쳤어요. 로컬 비즈니스와 Cafe24를 동시 지원하고, SQLite WAL 모드로 안정성을 높였어요. PyInstaller로 PEACH.exe 패키징까지 완료해서 실제 배포 가능한 상태예요.

PEACH 2
다음 단계

로컬 설치형 → SaaS 전환

Flask-Login 멀티유저 인증, ngrok 베타, Railway 이전까지 설계가 완료된 상태예요. 베타 체험단 5~10명을 모집해서 실사용 데이터를 쌓을 계획이에요.

Build

로컬 프로덕트

PEACH.exe를 실행하면 Flask 대시보드가 브라우저에서 바로 열려요. 데이터는 전부 로컬 SQLite에 저장되고, 외부 서버로 업체 정보나 메일 내용이 나가지 않아요. PyInstaller로 Windows/Mac 패키징까지 해뒀어요.
