01 · Origin

아이디어를 찾은 게 아니라, 실제 작업에서 계속 막히던 지점을 떼어냈습니다.

tomob의 Three.js 인트로를 만들 때 카메라와 오브젝트 위치를 AI에게 문장으로 전달했습니다. “조금 더 가까이”, “왼쪽으로”, “각도를 낮게” 같은 피드백이 반복됐습니다. 화면을 직접 보며 조정하면 몇 초면 끝날 일이었습니다.

02 · Existing Proof

필요한 기능부터 실제 제작 화면 안에 만들었습니다.

03 · Product Definition

Scene Director는 모델링 툴이 아니라 웹 연출을 완성하는 마지막 마일 도구입니다.

Blender를 대체하지 않습니다. GLB/GLTF 에셋이 준비된 다음, 웹페이지 안에서 카메라와 인터랙션을 조합하고 Three.js 설정으로 내보내는 구간에 집중합니다.

ASSET

3D 에셋

GLB/GLTF 드래그앤드롭, Scene Hierarchy, 표시·숨김·복제·삭제.

TRANSFORM

직접 조정

위치·회전·크기를 조절하고 장면 안에서 결과를 바로 확인.

CAMERA

카메라 포인트

시작점과 종료점을 저장하고 기본 무빙 프리셋을 연결.

INTERACTION

인터랙션

Hover, Click focus, Scroll reveal, Scroll camera를 프리셋으로 적용.

VFX

VFX 프리셋

Bloom, God ray, Portal, Ripple, Fog transition을 검증된 설정으로 제공.

EXPORT

코드 연결

Scene JSON과 Three.js 적용 코드를 확인하고 복사.

06 · MVP Boundary

범용 3D 에디터가 되지 않도록, 웹 장면 제작에 필요한 범위만 남겼습니다.

BUILD

실제로 구현할 것

에디터 UI, Three.js Preview, 샘플 에셋, Transform, 카메라 포인트, 프리셋 1종, JSON 출력.

DEMO ONLY

시연으로 남길 것

계정, 클라우드 저장, 협업, 전체 프로젝트 생성, 실제 AI API 연동.

좋은 AI 제품은 기능을 상상하는 데서가 아니라, 이미 반복해서 겪고 있는 마찰에서 시작한다고 생각했습니다.

07 · Validation Plan

멋있어 보이는 데모보다 실제 제작 시간이 줄었는지 확인합니다.

TIME

프롬프트 반복 방식과 직접 조정 방식의 완료 시간을 비교합니다.

TRANSFER
