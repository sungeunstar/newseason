PESTOMAP

비개발자 GIS 실무자의 반복 업무를 AI로 없앤다

Role PM · Developer
Type QGIS Plugin · AI Tool
Stack Python · PyQGIS · Gemini API
Duration 2026. 05 · MVP → 반복 개선 → v0.3.5

Background

GIS 실무자의 하루는 반복으로 채워져 있었어요

도시계획·감정평가 분야에서 일하는 지인이 있어요. QGIS를 매일 쓰는데, 하는 일의 상당 부분이 같은 작업의 반복이에요. shp 파일을 불러오고, 레이어마다 색상을 맞추고, 스케일바·방위표를 붙이고, PDF로 출력하는 일. 데이터가 바뀔 때마다, 프로젝트가 바뀔 때마다 처음부터 다시 해야 했어요.

이미 AI를 업무에 적극 쓰고 있는 사람이었어요. ChatGPT 프롬프트를 직접 구조화해서 법령 해석에 쓸 정도였고, 도구를 만들어주면 직접 쓸 수 있겠다는 판단이 섰어요.

Problem

매번 같은 세팅을 반복하는 시간이 아깝고, 실수도 생겨요. 한 번 설정해두면 자동으로 되면 좋겠어요.

Decision 01

먼저 할 수 없는 것을 확인했어요

처음 요청은 4가지였어요. 등기정보광장 API 자동화, QGIS 자동화, GPX 3D 영상 생성기, 용적률 계산 시트. 전부 한꺼번에 가면 아무것도 제대로 만들 수 없어요.

등기정보광장 API를 먼저 직접 조사했어요. 소유자 정보는 개인정보 보호 이유로 공공 API로 제공이 불가능했고, 셀레니움으로 우회하면 건당 수수료가 발생해 실익이 없었어요. 가능성이 없는 항목은 명확하게 제외하고, 실제로 반복 업무가 발생하는 QGIS 자동화에 집중했어요.

❌ 제외
등기정보광장 API
소유자 정보는 개인정보 사유로 공공 API 미제공. 셀레니움 우회 시 건당 수수료 발생 → 실익 없음. 스코프에서 제거.
✅ MVP 집중
QGIS 자동화 플러그인
실무에서 매일 쓰는 반복 작업. 즉각적인 시간 절감 효과. 사용자가 QGIS 실사용자라 진입 장벽 없음.

Decision 02

코드 없이 쓸 수 있어야 했어요

QGIS 플러그인 구현 방식은 두 가지였어요. Python 스크립트를 직접 실행하는 방식과, UI 패널로 동작하는 커스텀 플러그인 방식. 전자는 개발 지식이 없으면 쓰기 어려워요.

사용자가 "직접 해보고 싶다"고 했을 때, 그 직접은 코드 수정이 아니라 버튼을 클릭하는 것이어야 했어요. ZIP 설치 한 번으로 QGIS 패널에 바로 나타나는 플러그인으로 만들었어요.

여기에 Gemini AI 채팅을 연결했어요. "용도지역 레이어 색상을 파란 계열로 바꿔줘"처럼 자연어로 요청하면 실행되는 방식이에요. 메뉴 구조를 외우지 않아도 돼요.

Iteration

쓰면서 발견한 문제를 계속 고쳐나갔어요

핵심 기능만 넣은 첫 버전을 먼저 냈어요. 실제로 설치해서 쓰다 보면 설계 단계에서 보이지 않던 문제가 나와요. 버전마다 문제를 발견하고, 원인을 파악하고, 고치는 과정을 반복했어요.

v0.1
Feature
shp 자동 로드 · 색상 프리셋 · 출력
핵심 기능 3가지만 넣고 먼저 냈어요. ZIP 설치 후 QGIS 패널에서 바로 동작하는 걸 확인했고, 폴리곤 geometry 감지 버그를 발견해서 바로 v0.1.1로 패치했어요.
v0.3.0
Feature
Gemini AI 채팅 · 롤백 · 세션 기록
자연어로 레이어를 조작하는 AI 채팅을 넣었어요. 실수를 되돌리는 10단계 롤백, 작업 기록을 남기는 세션 로그도 같이 추가했어요.
QThread에서 AI 응답을 처리하다가 QGIS 크래시가 발생했어요. 원인을 파악하고 자동 분석 기능을 제거해 수동 요청 방식으로 바꿨어요. 크래시가 없어졌어요.
v0.3.2
Bug Fix
Gemini 429 에러 대응
Gemini API 호출 시 429 에러가 계속 났어요. 자동 재시도 로직(3초·8초 backoff)을 추가했지만, 근본적인 해결은 아니었어요.
재시도로 증상을 줄이긴 했지만, 모델의 RPM 한도 자체가 낮다는 걸 파악했어요. 모델 교체로 이어졌어요.
v0.3.4
Key Decision
Gemini 모델 교체 — 2.0-flash → 2.5-flash-lite
gemini-2.0-flash는 2026년 6월 서비스 종료 예정이고, 무료 RPM이 5로 낮아서 실무 사용 빈도에서는 구조적으로 429가 날 수밖에 없었어요.
gemini-2.5-flash-lite로 교체했어요. RPM 15, 일 1,000회 무료로 429가 근본적으로 해결됐어요. 기존 backoff 로직은 안전망으로 그대로 뒀어요.
v0.3.5
Feature
색상 설정 UI 팝업 추가
색상을 바꾸려면 config.json을 직접 열어서 수정해야 했어요. 비개발자에게 파일 수정을 요구하는 건 맞지 않아서, UI 팝업(QColorDialog)으로 바꿨어요. 저장하면 바로 config.json에 반영돼요.

Stack & Architecture

선택한 기술과 이유

QGIS 플러그인은 Python으로만 만들 수 있어요. PyQGIS API로 레이어 조작, 레이아웃 출력, UI 구성까지 전부 처리했어요. AI 연동은 REST API 직접 호출 방식으로 구현했어요. 외부 SDK를 추가하면 환경마다 설치 문제가 생길 수 있어서, 의존성을 최소화했어요.

Reflection

프로젝트를 통해 확인한 것들

범위 설정

요청 4가지 중 기술적으로 불가능한 것을 먼저 확인하고 제거했어요. 가능한 것에 집중하는 게 더 빠른 가치를 만들어요. "이건 안 되는 이유"를 명확하게 설명하는 것도 기획자의 역할이에요.

AI 운영

429 에러에 재시도 로직으로 대응했지만, 모델의 RPM 한도가 근본 원인이었어요. 증상을 고치는 것과 원인을 고치는 것은 달라요. 모델 sunset 일정을 확인하고 선제적으로 교체한 게 반복되는 문제를 없앴어요.

사용자 맥락

같은 기능이라도 사용자에 따라 전달 방식이 달라야 해요. 개발자라면 스크립트 실행으로 충분하지만, 이 사용자에게는 패널과 버튼이 있어야 했어요. UI 결정을 내릴 때마다 "이 사람이 코드 없이 쓸 수 있는가"를 기준으로 삼았어요.
