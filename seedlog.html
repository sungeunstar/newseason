Overview

농업 현장의 데이터를
기록하고 패턴으로 보는 도구

SeedLog는 수직농장을 직접 운영하면서 생긴 불편함에서 시작됐어요. 재배 기록, 환경 데이터, 수확량을 한 곳에서 보고 싶었는데, 딱 맞는 도구가 없었어요.

기획, 디자인, 개발, 배포까지 혼자 했어요. 0에서 1까지 만든 개인 프로젝트예요.

Overview 앱 메인 화면

Problem

농장 데이터는 흩어져 있고,
패턴을 보기가 어려워요

재배 일지는 메모장에, 환경 데이터는 엑셀에, 수확량은 머릿속에 있었어요. 데이터가 쌓이긴 했는데, 어디에 뭐가 있는지 찾기도 어렵고, 한눈에 연결해서 볼 수가 없었어요.

데이터가 없는 게 문제가 아니었어요. 데이터를 한 곳에 쌓고, 연결해서 볼 수 있는 도구가 없었던 거예요.

Features

핵심은 '다음에 같은 걸 심을 때
비교할 수 있는가'였어요

작물별 재배 기록

파종일, 발아일, 수확일, 수확량을 작물별로 남겨요. '지난번에 언제 파종했더라'를 찾는 게 시작이었어요. 이걸 남기지 않으면 매 시즌이 처음 시작이에요.

환경 데이터 추적

온도, 습도, EC, pH를 날짜별로 기록하고 그래프로 확인해요. 수치를 쌓는 게 목적이 아니에요. '수확량 좋았던 날 환경이 어땠지'를 역으로 찾는 게 목적이에요.

대시보드

지금 어느 작물이 어느 단계인지, 전체를 한 화면에서 봐요. 여러 작물을 동시에 재배할 때 어느 하나를 놓치지 않기 위해서예요.

처음부터 다 넣지 않았어요

뭐가 필요한지 몰랐어요. 직접 써보면서 필요한 기능을 하나씩 추가했어요. 지금 구조는 3번 바꾼 결과예요. 만들고 써봐야 뭐가 부족한지 보여요.

Feature 01 작물 기록 / 대시보드
Feature 02 환경 데이터 그래프

Build

스택 선택부터 배포까지
직접 했어요

Next.js + Supabase + Vercel로 직접 빌드했어요. 기획만 하는 PM이 아니라, 스스로 만들고 운영할 수 있다는 걸 보여주고 싶었어요. 스택을 고르는 것도 직접 했고, 배포 후 실제 농장에서 쓰면서 계속 고치고 있어요.

App Screenshots 실제 빌드 화면

Outcome

데이터가 쌓이면서
처음 몰랐던 게 보이기 시작했어요

농장에서 직접 쓰기 전까지는 어떤 데이터가 필요한지 몰랐어요. 써보면서 "이 작물은 EC가 올라가기 전에 수확량이 떨어지네"처럼 이전엔 막연하게 느꼈던 것들이 숫자로 보이기 시작했어요.

도구를 만드는 것보다 계속 쓰는 게 더 어려워요. SeedLog는 지금도 농장에서 실제로 쓰고 있어요. 쓰면서 바뀐 기능이 처음 설계한 것보다 더 유용한 경우가 많았어요.
